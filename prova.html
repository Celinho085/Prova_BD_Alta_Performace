Para que um banco de dados seja bem projetado e compreendido, é crucial documentar suas estruturas, regras e componentes. Essa documentação utiliza um conceito fundamental que descreve os dados, em vez de representar os dados de negócio em si. Como o material define o conceito de "metadados"?

4. (Análise de Sentenças)

A normalização é um processo técnico e formal introduzido por E. F. Codd, fundamental para a criação de bancos de dados relacionais robustos e eficientes. Seu principal objetivo é "purificar" a estrutura das entidades para evitar problemas conhecidos como anomalias de atualização. Com base no material, analise as sentenças a seguir sobre os problemas que a normalização busca resolver:

I- A normalização visa eliminar a redundância de dados, como ter que repetir o nome e a titulação de um professor para cada aluno matriculado em sua disciplina.
II- Um dos problemas evitados pela normalização é a perda acidental de informações, que ocorre, por exemplo, ao excluir o único aluno de uma disciplina e, com isso, perder os dados da própria disciplina e do professor.
III- A normalização corrige anomalias de inserção, que se manifestam quando não é possível adicionar um novo curso ao banco de dados até que ao menos um aluno esteja matriculado nele.

5. (Análise de Sentenças)

No modelo relacional, as chaves são atributos ou conjuntos de atributos essenciais para identificar registros de forma única e para estabelecer vínculos lógicos entre as tabelas. A partir dos conceitos apresentados, avalie as seguintes afirmativas:

I- Uma chave primária pode ser composta por mais de um atributo (chave composta), sendo utilizada quando um único atributo não é suficiente para garantir a unicidade de um registro.
II- Uma chave estrangeira é um atributo em uma tabela que faz referência à chave primária de outra tabela, podendo ter seus valores repetidos para representar relacionamentos do tipo "1 para N".
III- Uma superchave é um conjunto de atributos que identifica unicamente um registro, e quando essa superchave não pode ser mais reduzida, ela se torna uma chave estrangeira.

6. (Análise de Sentenças)

A cardinalidade é um conceito que define a natureza dos relacionamentos entre entidades, especificando quantos registros de uma entidade podem se associar a registros de outra. Com base no material, analise as proposições:

I- Em um relacionamento "1 para 1", um único registro em uma primeira entidade se relaciona com, no máximo, um único registro em uma segunda entidade.
II- Para modelar um relacionamento "N para M" (muitos-para-muitos), é necessário criar uma entidade intermediária, que conterá as chaves primárias de ambas as entidades originais como chaves estrangeiras.
III- Em um relacionamento "1 para N", a chave primária da entidade do lado "1" não pode se repetir em sua própria tabela, mas pode aparecer várias vezes como chave estrangeira na tabela do lado "N".

7. (Asserção e Razão)

Asserção (A): Ao projetar um banco de dados de alto desempenho, a prática recomendada é evitar o armazenamento de colunas calculadas, como "Valor Total da Venda".
PORQUE
Razão (R): O armazenamento de dados calculados representa um consumo desnecessário de recursos de disco, uma vez que esse valor pode ser gerado dinamicamente durante uma consulta via SQL, garantindo economia de espaço e consistência da informação.

8. (Asserção e Razão)

Asserção (A): Para uma tabela estar na Terceira Forma Normal (3FN), um dos requisitos é que não existam dependências funcionais transitivas entre seus atributos não-chave.
PORQUE
Razão (R): Uma dependência funcional transitiva ocorre quando um atributo não-chave depende diretamente de outro atributo não-chave, em vez de depender exclusivamente da chave primária da tabela.

9. (Asserção e Razão)

Asserção (A): O Dicionário de Dados é um produto da fase de análise de requisitos e deve conter informações como nomes de tabelas, nomes e tipos de colunas, e detalhes sobre chaves primárias e estrangeiras.
PORQUE
Razão (R): O objetivo principal do Dicionário de Dados é armazenar os próprios dados de negócio da empresa, como a lista de clientes ou o histórico de transações, para garantir que eles sejam acessados de forma rápida e eficiente.

Parte 2 – Unidade 02 (MySQL & SQL)

1. (Múltipla Escolha)

A linguagem SQL é organizada em subconjuntos de comandos, cada um com uma finalidade específica dentro do gerenciamento de um banco de dados, seja para definir estruturas, manipular dados ou controlar permissões. Essa organização ajuda a compreender o papel de cada operação. Qual dos comandos a seguir pertence à Linguagem de Definição de Dados (DDL - Data Definition Language)?

2. (Múltipla Escolha)

Ao criar tabelas, especialmente colunas que servirão como chave primária, os SGBDs oferecem mecanismos para automatizar a geração de valores únicos. Isso simplifica a inserção de dados e garante que cada registro tenha um identificador exclusivo sem intervenção manual. Qual é a principal finalidade da propriedade AUTO_INCREMENT no MySQL?

3. (Múltipla Escolha)

Em um banco de dados relacional normalizado, as informações frequentemente se encontram distribuídas em várias tabelas. Para gerar relatórios úteis, é preciso combinar os dados dessas tabelas por meio de consultas, utilizando cláusulas de junção. Qual tipo de JOIN deve ser utilizado quando se deseja retornar apenas os registros que possuem correspondência em ambas as tabelas que estão sendo relacionadas?

4. (Análise de Sentenças)

A Linguagem SQL é dividida em subconjuntos para organizar suas funcionalidades, sendo a distinção entre DDL (Data Definition Language) e DML (Data Manipulation Language) uma das mais fundamentais. A DDL trata da estrutura dos objetos, enquanto a DML opera sobre os dados contidos neles. Com base nos conceitos do material, analise as seguintes sentenças:

I- Os comandos UPDATE e DELETE, utilizados para, respectivamente, alterar e remover registros de uma tabela, são classificados como parte da DML.
II- O comando ALTER TABLE, que permite modificar a estrutura de uma tabela (adicionando ou removendo colunas), é um exemplo de comando DDL.
III- A principal diferença entre os comandos é que a DML afeta o dicionário de dados do banco, enquanto a DDL afeta apenas os registros das tabelas.

5. (Análise de Sentenças)

Além das tabelas permanentes que compõem o modelo de dados de um sistema, o MySQL permite a criação de tabelas temporárias, um recurso valioso para desenvolvedores durante a manipulação e análise de dados complexos. A respeito das tabelas temporárias, conforme descrito no material, avalie as seguintes afirmativas:

I- Uma tabela temporária é exclusiva da sessão do usuário que a criou e deixa de existir automaticamente assim que o usuário se desconecta do SGBD.
II- É possível criar uma tabela temporária já populada com os dados resultantes de um comando SELECT, o que facilita o armazenamento de resultados de consultas complexas.
III- Os comandos de manipulação de dados (DML) como INSERT, UPDATE e DELETE não podem ser utilizados em tabelas temporárias, apenas em tabelas permanentes.

6. (Análise de Sentenças)

Ao construir uma consulta que envolve múltiplas tabelas, a escolha da cláusula JOIN é determinante para o conjunto de resultados. LEFT JOIN e RIGHT JOIN são particularmente úteis para encontrar dados que podem não ter uma correspondência em uma das tabelas. Com base nos exemplos e explicações do material, analise as proposições:

I- Ao usar SELECT ... FROM tbcliente_compra LEFT JOIN tbproduto ..., o resultado incluirá todos os clientes, e para aqueles que não compraram um produto, o campo do produto será exibido como NULL.
II- O resultado de uma consulta com RIGHT JOIN sempre trará a mesma quantidade de linhas que uma consulta com INNER JOIN, apenas exibindo as colunas em uma ordem diferente.
III- O FULL JOIN combina os resultados de um LEFT JOIN e um RIGHT JOIN, garantindo que todos os registros de ambas as tabelas (esquerda e direita) apareçam no resultado final, com NULL onde não houver correspondências.
